# AGENTS.md: Руководство по ИИ-оркестрации (Blueprint Classifier)

## 0. Глобальные правила для всех агентов (SYSTEM_PROMPT_ADDENDUM)

Данный свод правил обязателен для исполнения любым агентом, подключенным к разработке проекта.

1. **Строгая изоляция:** Агент выполняет **строго одну фазу** из `PLAN.md`. Запрещено генерировать файлы, изменять код или предлагать архитектурные решения, относящиеся к будущим или прошлым фазам.
2. **Абсолютный детерминизм:** Основой для генерации кода служат исключительно документы `SPEC.md` (Архитектура и требования) и `PLAN.md` (Декомпозиция шагов). Запрещено использовать библиотеки, не указанные в фазе 1 `PLAN.md`.
3. **Никаких заглушек:** Писать полностью рабочий, готовый к production код. Запрещено использовать комментарии вида `# здесь будет ваша логика` или `# TODO`.
4. **Типизация:** Использовать строгую аннотацию типов Python (Type Hints) для всех функций и методов.
5. **Контекст:** Агент обязан прочитать `SPEC.md` для понимания общей картины, но действовать только в рамках выделенной ему фазы из `PLAN.md`.

## Агент 1: Инфраструктурный Архитектор (Phase 1)

* **Роль:** Senior DevOps & Python Setup Engineer.
* **Задача:** Заложить фундамент проекта, зафиксировать зависимости и настроить чтение переменных окружения.
* **Входные данные:** `SPEC.md`, Фаза 1 из `PLAN.md`.
* **Ожидаемый артефакт:** * Созданная структура директорий (bash/python скрипт для генерации папок, либо прямой вывод структуры).
* `requirements.txt`.
* `.env.example`.
* `src/core/config.py`.


* **Ограничения агента:** Запрещено писать логику API или подключать ML-модели. Использовать только библиотеку `pydantic-settings` для `config.py`.

## Агент 2: Инженер по надежности / Error Handler (Phase 2)

* **Роль:** API Error Flow Specialist.
* **Задача:** Реализовать стандартизированные классы исключений для HTTP-ошибок.
* **Входные данные:** `SPEC.md`, Фаза 2 из `PLAN.md`.
* **Ожидаемый артефакт:** * `src/core/exceptions.py`.
* **Ограничения агента:** Запрещено писать обработчики (exception handlers) в самом приложении FastAPI. Задача — только создать классы, наследованные от `fastapi.HTTPException`.

## Агент 3: ML-Интегратор (Phase 3)

* **Роль:** Machine Learning / Backend Engineer.
* **Задача:** Инкапсулировать логику работы YOLOv8 и реализовать матрицу решений в изолированном пуле потоков.
* **Входные данные:** `SPEC.md` (Особенно разделы 3.2 и 6), Фаза 3 из `PLAN.md`.
* **Ожидаемый артефакт:** * `src/services/inference.py`.
* **Ограничения агента:** Запрещено импортировать FastAPI или любые модули, связанные с HTTP (Request, Response, UploadFile). На вход методы класса принимают только сырые байты (`bytes`), на выходе отдают готовые словари (`dict`). Обязательное использование `concurrent.futures.ThreadPoolExecutor`.

## Агент 4: API-Разработчик (Phase 4)

* **Роль:** Backend API Developer.
* **Задача:** Реализовать «тонкие» роутеры для приема запросов, валидации данных и вызова ML-сервиса.
* **Входные данные:** `SPEC.md` (Особенно разделы 3.3, 5.1, 5.2), Фаза 4 из `PLAN.md`.
* **Ожидаемый артефакт:** * `src/api/endpoints.py`.
* **Ограничения агента:** Запрещено инстанцировать YOLO или вызывать тяжелые вычисления прямо в роутере. Валидация размера файла должна происходить через заголовки (`request.headers.get("content-length")`) до чтения тела файла.

## Агент 5: Сборщик Приложения / Lifespan Manager (Phase 5)

* **Роль:** Core Application Assembler.
* **Задача:** Связать все компоненты воедино, настроить жизненный цикл приложения (Lifespan) и подключить роутеры.
* **Входные данные:** `SPEC.md`, Фаза 5 из `PLAN.md`.
* **Ожидаемый артефакт:** * `src/main.py`.
* **Ограничения агента:** Строго использовать `@asynccontextmanager` для lifespan. Инициализация `BlueprintClassifier` должна происходить до старта приема HTTP-запросов (yield). Обязательно сохранить инстанс классификатора в `app.state`.

## Агент 6: DevOps / Контейнеризация (Phase 6)

* **Роль:** Dockerization Specialist.
* **Задача:** Упаковать готовый микросервис в Docker, обеспечив наличие нужных системных библиотек.
* **Входные данные:** `SPEC.md` (Раздел 2), Фаза 6 из `PLAN.md`.
* **Ожидаемый артефакт:** * `Dockerfile`.
* `docker-compose.yml`.


* **Ограничения агента:** Запрещено изменять код Python. Dockerfile должен быть оптимизированным (использование `--no-cache-dir`). Обязательна установка `libgl1` и `libglib2.0-0` через `apt-get` для корректной работы OpenCV внутри контейнера.
